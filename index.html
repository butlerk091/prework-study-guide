HTML

  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta attribute contains information about the page that is used by the browser.
  • HTML documents aways begin with - !doctype html - inbetween angle brackets.
  • Declarations are not case sensitive; Elements are case sensitive.
  • HTML attribute: defines additional info about an element.
  • Element: refers to the entire element , opening tag to closing tag.
  • Tag: refers to what only appears inside the angle brackets.
  • Semantic HTML Elements: the name of the element describes its content and role it plays on a webpage.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Github: web interface for Git, a version control software, that allows multiple people to make seperate changes to a program at the same time (cloud servie).
  • Git: a version control system that lets you manage and keep track of source code history.
  • Repository(repo): a folder for your project that contains all of the files and their revision history.
  • Branch: Workspace where you can make changes that don't affect the live state.
  • Markdown(.md): The way content is formatted on Github.
  • Commit Changes: Whenever you add, edit, or delete a file, you're making a commit.
  • Pull Request(PR): Asking for changes on a branch to merge into another branch.
  • Merge: When a PR is approved, changes are pulled in or merged into main branch.
  • Issue: Way to report new tasks or content fixes.
  • Federalist: a platform that securely deploys a website from Github repo in minutes and lets users preview proposed and published changes.
  • Main Branch: should be the final, most updated version of the application.
  • Work done on applicationsshould be done on feature branches through commits.
  • ALWAYS stage after saving when applicable.
  • git: must precede every Git statement.
  • IDE (Integrated Development Environment): Software for building applications that combines common developer tools into a single GUI.
  • git pull origin main: command allows user to update main branch from remote repo to local repo. Important to check that both repos are synced prior to updating remote repo
  • git push origin [feature/feature-branch-name]: command send updated version of that feature branch to remote repo from local repo in order to do a PR at Github.
  • git add -A: adds all changes in current branch to the staging area; stages a branch.
  • git commit -m "brief description for commit": command submits commit within current branch.
