HTML

  • HTML elements work together to build the structure of a webpage and provide a framework that makes it easy for a user to navigate the webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta attribute contains information about the page that is used by the browser.
  • HTML documents aways begin with - !doctype html - inbetween angle brackets.
  • Declarations are not case sensitive; Elements are case sensitive.
  • HTML attribute: defines additional info about an element.
  • Element: refers to the entire element , opening tag to closing tag.
  • Tag: refers to what only appears inside the angle brackets.
  • Semantic HTML Elements: the name of the element describes its content and role it plays on a webpage.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • CSS defines colors, aesthetics, visual layout, and orginization of HTML.
  • Three ways to style:
    • CSS Inline
    • CSS Internal Style Sheet
    • CSS External Style Sheet
  • inline CSS: when an element is directly styled in an HTML file.
  • Inline styling must be edited individually whenever there are design changes.
  • Internal Stying Sheet: contains CSS rules for the webpage in the head section of the HTML file.
  • External Styling Sheet: All CSS rules are stored in a separate file.
    • Makes design changes easier.
    • Most common way to style a webpage.
  • style: attribute tha makes changes to style for HTML files.
  • Separations of Concern: Seperating different files based on use.
    • Based on the idea that each section of code should have its own responsibility.
  • Selector: defines the element/attributes to which the rules/declarations will apply.
  • Declarations: contains the CSS PROPERTY we want to apply and the VALUE of the property.
  • margin: property indicates how much space we want around the outside of an element.
  • padding: propert adds space around the inside of an element.
  • Every HTML element = box; each box has the following:
    • margins outside the element
    • borders
    • padding
    • content
  • * symbol: used to apply a rule to ALL elements visable on a webpage.
  • Two or more elements sharing the same styling can be grouped together as selectors, with a comma to separate each one. ex. header, footer
  • text-align: property centers an element on a webpage.
  • img: selector assigns property values to any image element in the markup.
  • link: element creates the connection to an external file in HTML.
  • rel: attribute specifies the relationship between the current document and the linked document resource.
  • href:attribute specifies location (URL) of the external resource.
  • Class Attributes: allows us to share a CSS rule to certain, but not all, elements on an HTML file.
  • box-shadow: property adds shadow effects around an elements frame. Consists of the following attributes:
    • x & y offsets
    • speed & blur radius
    • color
  • Colors can be selected semantically or by hexadecimal code.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Github: web interface for Git, a version control software, that allows multiple people to make seperate changes to a program at the same time (cloud servie).
  • Git: a version control system that lets you manage and keep track of source code history.
  • Repository(repo): a folder for your project that contains all of the files and their revision history.
  • Branch: Workspace where you can make changes that don't affect the live state.
  • Markdown(.md): The way content is formatted on Github.
  • Commit Changes: Whenever you add, edit, or delete a file, you're making a commit.
  • Pull Request(PR): Asking for changes on a branch to merge into another branch.
  • Merge: When a PR is approved, changes are pulled in or merged into main branch.
  • Issue: Way to report new tasks or content fixes.
  • Federalist: a platform that securely deploys a website from Github repo in minutes and lets users preview proposed and published changes.
  • Main Branch: should be the final, most updated version of the application.
  • Work done on applications should be done on feature branches through commits.
  • ALWAYS stage after saving when applicable.
  • git: must precede every Git statement.
  • IDE (Integrated Development Environment): Software for building applications that combines common developer tools into a single GUI.
  • git pull origin main: command allows user to update main branch from remote repo to local repo. Important to check that both repos are synced prior to updating remote repo
  • git push origin [feature/feature-branch-name]: command send updated version of that feature branch to remote repo from local repo in order to do a PR at Github.
  • git add -A: adds all changes in current branch to the staging area; stages a branch.
  • git commit -m "brief description for commit": command submits commit within current branch.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • statement: piece of code that tells th computer to do something.
  • JavaScript is case sensitive.
  • Object: could be anything.
  • array: structure that allows you to store multiple values within a single reference.
  • Event: code structures that listen for activity in the browser, and run code in response.
  • var: used to declare a variable.
  • index: a unique number that identifies the location for a data item within an array.
  • JavaScript arrays are zero-indexed and sequential.
  • A function is a set of instructions that tells the computer how to perform a task.
  • There are two requirements to use a function:
    1. Define the function
    2. Call the funtion
  • A functions name is always followed by parenthesis ().
  • Code Block, a.k.a function body, is the set of instructions within a function.
  • Control flow is the order in which a computer executes code in a script.
